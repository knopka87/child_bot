### Проблемы и решения

#### 1. Ошибка 500 при обработке изображения (SQL Scan error)

**Проблема:**
```
POST /attempts/{id}/process → 500 Internal Server Error
sql: Scan error on column index 6, name "detect_result":
unsupported Scan, storing driver.Value type <nil> into type *json.RawMessage
```

**Причина:**
JSONB колонки в PostgreSQL могут содержать NULL, но `json.RawMessage` не поддерживает NULL при SQL сканировании.

**Решение:**
```go
// Файл: /api/internal/store/attempt.go:26-45

// Было:
DetectResult  json.RawMessage
ParseResult   json.RawMessage
HintsResult   json.RawMessage
CheckResult   json.RawMessage

// Стало:
DetectResult  []byte  // NULL → пустой слайс ✅
ParseResult   []byte
HintsResult   []byte
CheckResult   []byte
```

---

#### 2. Mock данные вместо реальных подсказок от LLM

**Проблема:**
Фронтенд всегда показывал: *"Это тестовая подсказка. Бэкенд ещё не реализован (Phase 4)."*

**Причина:**
Fallback оператор `||` в `getResult` заменял реальные данные на mock.

**Решение:**
```typescript
// Файл: /frontend/src/api/help.ts:101-134

// Было:
hints: response.result?.hints || [
  { id: '1', content: 'Это тестовая подсказка...' },
  // mock данные ❌
]

// Стало:
hints: response.result?.hints || []  // Только реальные данные ✅
```

```
1. POST /attempts → создание attempt (status: created)

```

**Изменения:**

- `frontend/src/types/help.ts`: добавлен `CreateAttemptResponse`
- `frontend/src/types/check.ts`: добавлен `CreateCheckAttemptResponse`

---

#### 2. Ошибка загрузки изображения (400: invalid character '-' in numeric literal)

**Было:**
- Отправка файла как `multipart/form-data`
- Бэкенд ожидал JSON с base64

**Стало:**
```javascript
POST /attempts/{id}/images
{
  "image_type": "task",     // "task" или "answer"
  "image_data": "base64..." // base64-encoded изображение
}
```

**Изменения:**
- `frontend/src/api/help.ts`: конвертация Blob → base64
- `frontend/src/api/check.ts`: аналогичная конвертация
- Размер сжатия уменьшен до 7 МБ (чтобы base64 не превысил лимит 10 МБ)

---

#### 3. Лимит размера файла

**Проблема:**
- Бэкенд: лимит 10 МБ на JSON
- Base64 увеличивает размер на ~33%
- 10 МБ файл → ~13 МБ base64 → ❌ превышение лимита

**Решение:**
```javascript
// UploadPage.tsx
const maxSizeMB = 7; // После base64: ~9.3 МБ < 10 МБ ✅
```
