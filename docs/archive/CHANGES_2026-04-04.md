## 2. Исправление ошибки 401 при email verification

### Проблема
```
POST http://localhost:8080/email/verify/send 401 (Unauthorized)
{status: 401, message: 'Missing X-Child-Profile-ID header'}
```

Email verification эндпоинты требовали `X-Child-Profile-ID` заголовок, но на этапе онбординга профиль ещё не создан.

### Решение

#### Auth Middleware
- **Файл:** `api/internal/api/middleware/auth.go`

**Изменение 1:** Добавлены public paths
```go
// Строка 87-88 (добавлено)
"/legal/",           // Legal documents доступны всем
```

**Изменение 2:** Email endpoints не требуют childProfileID
```go
// Строка 106 (добавлено)
"/email/",         // Email verification - часть onboarding, до создания профиля
```

### Тест
```bash
curl -X POST http://localhost:8080/email/verify/send \
  -H 'Content-Type: application/json' \
  -H 'X-Platform-ID: web' \
  -d '{"email":"test@example.com","parentUserId":"test-user-123"}'

# Результат: 200 OK
{"devCode":"140549","expiresAt":"...","message":"Verification code sent to email"}
```

### Статус
✅ **Исправлено и протестировано**
- Email verification работает без `X-Child-Profile-ID`
- Legal documents доступны без авторизации
- Backend пересобран и перезапущен

## 3. Сохранение прогресса онбординга

### Функциональность

Теперь прогресс онбординга автоматически сохраняется и восстанавливается при перезагрузке страницы.

### Изменения

#### Storage Keys
- **Файл:** `frontend/src/lib/platform/vk-storage.ts`
- **Строки:** 124-131

**Добавлены новые ключи:**
```typescript
ONBOARDING_STEP: 'onboarding_step',
ONBOARDING_GRADE: 'onboarding_grade',
ONBOARDING_AVATAR: 'onboarding_avatar',
ONBOARDING_EMAIL: 'onboarding_email',
ONBOARDING_EMAIL_VERIFIED: 'onboarding_email_verified',
ONBOARDING_DISPLAY_NAME: 'onboarding_display_name',
ONBOARDING_CONSENTS: 'onboarding_consents',
```

#### OnboardingPageNew.tsx

**Изменение 1:** Восстановление прогресса при монтировании
- **Строки:** 48-118
- **Действие:**
  - Загрузка сохранённых данных из storage
  - Восстановление состояния (step, grade, avatar, email, consents)
  - Логирование для отладки

**Изменение 2:** Автоматическое сохранение при изменениях
- **Строки:** 121-163
- **useEffect:** Следит за всеми полями онбординга
- **Действие:** Сохраняет в storage при каждом изменении
- **Условие:** Не сохраняет на шаге "completed"

**Изменение 3:** Очистка после завершения
- **Строки:** 289-296
- **Действие:** Удаление всех временных данных после успешного создания профиля

### Сохраняемые данные

| Поле | Storage Key | Тип | Пример |
|------|-------------|-----|--------|
| Текущий шаг | `onboarding_step` | string | "email" |
| Класс | `onboarding_grade` | string | "3" |
| Аватар | `onboarding_avatar` | string | "cat" |
| Email | `onboarding_email` | string | "test@example.com" |
| Email verified | `onboarding_email_verified` | string | "true" |
| Имя | `onboarding_display_name` | string | "Саша" |
| Согласия | `onboarding_consents` | JSON | `{"adultConsent":true,...}` |

### Пользовательские сценарии

**Сценарий 1:** Прерывание на этапе email
```
1. Выбрать класс → аватар → email
2. [Перезагрузка]
3. → Восстанавливается на шаге email_verification
4. Ввести код → продолжить
```

**Сценарий 2:** Множественные перезагрузки
```
1. Каждый шаг → [Перезагрузка]
2. → Всегда восстанавливается корректное состояние
3. Продолжить с того же места
```

### Статус
✅ **Реализовано и готово к тестированию**
- Frontend компилируется без ошибок
- Логирование добавлено для отладки
- Создана документация и тест-план
