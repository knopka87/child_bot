## Архитектурные решения

### Почему храним прогресс в storage?

**Альтернативы:**
1. ❌ Не сохранять - плохой UX
2. ❌ Сохранять на сервере - требует API, сложнее
3. ✅ Storage (VK/localStorage) - просто, быстро, работает

**Выбрано:** Storage - оптимально для MVP

### Почему "Шаг X из Y" вместо "%"?

**Исследование:**
- 17% на первом шаге - сбивает с толку
- "Шаг 1 из 6" - понятно и информативно
- Пользователь видит сколько осталось

**Выбрано:** "Шаг X из Y" - лучше UX

### Почему схема валидации без автоматических параметров?

**Принцип:**
- Схема описывает что **требуется от пользователя**
- Автоматические параметры - внутренняя логика системы
- Разделение ответственности

**Выбрано:** Чистая схема без автоматики
