# Тестирование сохранения прогресса онбординга

## Быстрый тест

### Тест 1: Базовое сохранение и восстановление

1. Откройте http://localhost:5173 (или запустите VK Mini App)
2. Начните онбординг:
   - Выберите класс (например, 3)
   - Нажмите "Далее"
3. **Перезагрузите страницу** (F5 или Cmd+R)
4. ✅ **Ожидаемый результат**: Вы остаётесь на шаге выбора аватара, класс "3" сохранён

### Тест 2: Прогресс через несколько шагов

1. Продолжите онбординг:
   - Выберите аватар
   - Нажмите "Далее"
   - Введите email (test@example.com)
   - Нажмите "Далее"
2. **Перезагрузите страницу**
3. ✅ **Ожидаемый результат**:
   - Вы на шаге "Проверка email"
   - Email "test@example.com" сохранён
   - Класс и аватар также сохранены

### Тест 3: Email verification с перезагрузкой

1. На шаге "Проверка email":
   - Посмотрите код в консоли браузера (или в логах backend)
   - **НЕ ВВОДИТЕ КОД, перезагрузите страницу**
2. ✅ **Ожидаемый результат**:
   - Вы остались на шаге "Проверка email"
   - Email сохранён
   - Можете нажать "Отправить код повторно"
3. Введите полученный код → код работает

### Тест 4: Завершение и очистка

1. Пройдите весь онбординг до конца
2. Откройте DevTools → Application → Storage → Local Storage
3. ✅ **Ожидаемый результат**:
   - Ключи `onboarding_*` удалены
   - Остались только `profile_id`, `user_id`, `onboarding_completed`

## Проверка в DevTools

### Chrome/Edge DevTools

1. Откройте DevTools (F12)
2. Перейдите на вкладку **Application** → **Local Storage** → http://localhost:5173
3. Найдите ключи:
   - `onboarding_step` - текущий шаг
   - `onboarding_grade` - выбранный класс
   - `onboarding_avatar` - ID аватара
   - `onboarding_email` - введённый email
   - `onboarding_email_verified` - "true" или "false"

### Console Logs

В консоли браузера вы должны видеть логи:

```
[Onboarding] Progress saved: {step: 'avatar', grade: 3, avatarId: 'cat', ...}
[Onboarding] Restoring progress from storage: {step: 'avatar', grade: '3', ...}
```

## Backend логи

Для проверки email verification:

```bash
docker compose -f docker/docker-compose.dev.yml logs -f backend | grep Email
```

Вы увидите:
```
[EmailHandler] Verification code for test@example.com: 123456 (expires at ...)
```

## Сценарии тестирования

### Сценарий A: Прерывание на шаге email

1. Выбрать класс → аватар → ввести email
2. Перезагрузить страницу
3. Проверить, что вы на шаге email, данные сохранены
4. Нажать "Далее" → отправляется код
5. Перезагрузить страницу
6. Проверить, что вы на шаге verification
7. Ввести код → продолжить

### Сценарий B: Возврат через несколько часов

1. Начать онбординг, пройти 2-3 шага
2. Закрыть браузер
3. Открыть через несколько часов
4. Проверить, что прогресс сохранён
5. Продолжить с того же места

### Сценарий C: Множественные перезагрузки

1. После каждого шага делать перезагрузку страницы
2. Проверять, что каждый раз восстанавливается корректное состояние
3. Дойти до конца
4. Проверить, что временные данные удалены

## Troubleshooting

### Проблема: Прогресс не сохраняется

**Проверьте:**
1. Открыты ли DevTools → Console
2. Есть ли ошибки в консоли
3. Работает ли localStorage (не заблокирован в приватном режиме)
4. Логи: `[Onboarding] Progress saved` должны появляться

### Проблема: Прогресс не восстанавливается

**Проверьте:**
1. Логи: `[Onboarding] Restoring progress from storage` при загрузке
2. DevTools → Application → Local Storage → наличие ключей
3. Формат данных (должен быть строкой или JSON)

### Проблема: Email verification не работает после перезагрузки

**Проверьте:**
1. Срок действия кода (15 минут)
2. Логи backend: `[EmailHandler] Verification code for ...`
3. Email сохранён в storage: `onboarding_email`
4. Можете нажать "Отправить код повторно"
