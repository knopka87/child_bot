## 🔌 API интеграция

### Эндпоинт истории

```
GET /api/v1/attempts/history?child_profile_id={id}&limit=50&offset=0&filter=all
```

**Параметры:**
- `child_profile_id` - ID профиля ребёнка
- `limit` - количество записей (50)
- `offset` - смещение (0)
- `filter` - фильтр ('all', 'help', 'check')

**Ответ:**
```json
{
  "attempts": [
    {
      "attempt_id": "uuid",
      "mode": "help" | "check",
      "attempt_status": "completed" | "failed" | "cancelled",
      "result_status": "correct" | "has_errors" | "wrong",
      "error_count": 0,
      "thumbnail_url": "https://...",
      "scenario_type": "single_photo" | "two_photo",
      "created_at": "2026-04-04T10:00:00Z",
      "completed_at": "2026-04-04T10:05:00Z"
    }
  ],
  "total": 10,
  "has_more": false
}
```

### Эндпоинт деталей

```
GET /api/v1/attempts/{attemptId}/result
```

**Ответ с ошибками:**
```json
{
  "attempt_id": "uuid",
  "attempt_status": "completed",
  "result_status": "has_errors",
  "error_count": 2,
  "errors": [
    {
      "error_block_id": "err1",
      "step_number": 2,
      "line_reference": "Строка 3",
      "error_type": "calculation",
      "error_message": "Неправильное вычисление",
      "error_hint": "Проверьте знак операции",
      "location_type": "line"
    }
  ],
  "used_hints_count": 1
}
```

## 📊 Аналитика

### Новые события

| Событие | Когда срабатывает | Параметры |
|---------|-------------------|-----------|
| `history_opened` | Открытие страницы | `child_profile_id` |
| `history_back_clicked` | Клик "Назад" | `child_profile_id` |
| `history_item_clicked` | Клик на карточку | `child_profile_id`, `attempt_id`, `mode`, `status` |
| `history_filter_changed` | Смена фильтра | `child_profile_id`, `filter_type`, `filter_value` |
| `history_retry_clicked` | Клик "Повторить" | `child_profile_id`, `attempt_id`, `mode` |
| `history_fix_errors_clicked` | Клик "Исправить" | `child_profile_id`, `attempt_id` |

## 🎨 UX особенности

### Статусы попыток

| Статус | Иконка | Цвет | Текст |
|--------|--------|------|-------|
| `success` | ✓ CheckCircle | Зелёный | "Решено верно" |
| `error` | ✗ XCircle | Красный | "Есть ошибки" |
| `in_progress` | ⏱ Clock | Жёлтый | "В процессе" |

### Фильтры

**Режим:**
- Все
- Помощь
- Проверка

**Статус:**
- Все статусы
- Верно
- Ошибки

### Модальное окно

**Секции:**
1. Заголовок (режим, дата)
2. Бейдж статуса
3. Подсказки (если использовались)
4. Изображения (сетка)
5. Результат (summary)
6. Ошибки (список с описанием)
7. Действия (кнопки)

**Кнопки:**
- "Исправить и проверить" (для ошибок)
- "Повторить" (новая попытка)
- "Закрыть"

### 1. Запустить проект

```bash
cd /Users/a.yanover/Xsolla/child_bot
docker compose -f docker/docker-compose.dev.yml up
```

### 2. Открыть страницу

```bash
# 1. Перейти в профиль
open http://localhost:5173/profile

# 2. Кликнуть на "История"
# Должна открыться страница истории
```

### 3. Проверить функциональность

**Фильтрация:**
- Кликнуть "Помощь" → должны остаться только help попытки
- Кликнуть "Верно" → должны остаться только успешные

**Детали:**
- Кликнуть на карточку → должна открыться модалка
- Проверить все секции (статус, подсказки, ошибки)
- Кликнуть "Повторить" → переход на нужную страницу
- Кликнуть "Закрыть" → модалка закрывается

**Состояния:**
- Пустая история → текст "История пуста"
- Ошибка загрузки → кнопка "Повторить"
- Загрузка → спиннер
