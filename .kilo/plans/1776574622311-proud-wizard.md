# План разработки еженедельного отчёта для пользователей

## Цель
Разработать алгоритм генерации красивого HTML-отчёта с полезной информацией о пользователе за неделю, сохранить его в БД и обеспечить еженедельную отправку родителям.

## Анализ существующих данных
На основе исследования кода:
- **Модель пользователя**: `child_profiles` (имя, аватар, класс, уровень, XP, монеты, достижения, streak).
- **Активность**: `attempts` (попытки решения задач, время, подсказки, успехи).
- **Достижения**: `achievements` и `child_achievements` (разблокированные достижения).
- **Битвы**: `villain_battles` (прогресс в битвах с villain'ами).
- **Существующий сервис**: `WeeklyReportData` в `api/internal/service/report.go` уже агрегирует данные за неделю.

## Алгоритм генерации отчёта
1. **Сбор данных за неделю**: Для каждого пользователя агрегировать:
   - Профиль: имя, аватар, класс, уровень, XP, монеты, streak.
   - Активность: общее время, попытки (успешные/неудачные), подсказки, точность.
   - Достижения: новые разблокированные за неделю.
   - Битвы: прогресс в текущей битве, заработанные монеты.
   - Сравнение с предыдущей неделей: изменения в попытках, точности.

2. **Генерация HTML**: Использовать Go-шаблоны для создания красивого HTML с таблицами, графиками и стилями.

3. **Сохранение в БД**: Создать таблицу `weekly_reports` с полями: user_id, report_date, html_content, sent_at.

4. **Отправка**: Интегрировать с email-сервисом для отправки родителям.

## Шаги реализации
1. **Создать модель WeeklyReport** в БД (миграция).
2. **Расширить сервис report.go**: добавить функцию генерации HTML-отчёта.
3. **Добавить шаблон HTML**: создать файл шаблона для отчёта.
4. **Реализовать сохранение**: функция для сохранения отчёта в БД.
5. **Настроить scheduler**: cron-задача для еженедельной генерации и отправки.
6. **Интеграция с frontend**: обновить ParentReport для показа реальных отчётов.

## Требования к HTML-отчёту
- Красивый дизайн: цвета, шрифты, иконки.
- Разделы: профиль, активность, достижения, битвы, сравнение.
- Таблицы и графики для визуализации данных.

- Локализация: отчёт на русском?
- Безопасность: не раскрывать чувствительные данные.
