# Тестовые данные для E2E тестов

В этой директории должны находиться тестовые изображения для E2E тестирования.

## Как это работает

Тест `TestE2E_HintFlow` автоматически находит **все изображения** (*.jpg, *.jpeg, *.png) в этой директории и запускает полный цикл подсказок для каждого из них:

1. Устанавливает класс (3 класс)
2. Отправляет фото задачи
3. Ожидает распознавание (Detect + Parse)
4. Подтверждает результат распознавания
5. Запрашивает подсказки до исчерпания
6. Сохраняет результаты в `results/`

## Как добавить тестовые изображения

1. Сфотографируйте задачи из учебника математики
2. Сохраните файлы в эту директорию с понятными именами:
   - `testdata/task_math_grade3_page10.jpg`
   - `testdata/peterson_part1_task5.png`
   - `testdata/geometry_task.jpg`
3. Убедитесь, что изображения четкие и читаемые

## Примеры подходящих задач

- Простые арифметические выражения
- Текстовые задачи
- Геометрические задачи с рисунками
- Задачи с таблицами
- Задачи на логику

## Структура результатов

Для каждого изображения создается отдельный файл результатов:
```
results/
  task_math_grade3_page10_1708123456.json
  peterson_part1_task5_1708123789.json
```

## Запуск тестов

```bash
# Через Makefile (рекомендуется)
make test-e2e-setup   # Подготовить тестовую БД (один раз)
make test-e2e         # Запустить E2E тесты

# Или напрямую
export TEST_LLM_PROXY_URL="http://localhost:8081"
export TEST_DATABASE_URL="postgres://user:pass@localhost:5432/child_bot_test"
cd api
go test -v ./test/e2e/... -timeout 30m
```

## Примечания

- Каждое изображение тестируется как отдельный подтест
- Тесты пропускаются если директория пустая
- Timeout увеличен до 30 минут для множества изображений
- Результаты сохраняются даже при ошибках
